# Codex：使用 revert 回滚时，用户已编辑与源文件冲突怎么处理？

本文基于现有文档说明 **Codex 当前**是否具备“revert 回滚时发现用户已编辑、与源文件冲突”的流程与处理。只讲现状，不讨论未来可如何实现。

---

## 1) 结论

**当前文档中 Codex 与回滚相关的能力是“回滚最近 N 个用户回合”（如 `drop_last_n_user_turns()`），属于对话历史的回退，未描述文件级的 revert/rollback。**  
因此“revert 时发现用户已经编辑过、与源文件冲突怎么处理”的**文件冲突**场景，在现有文档范围内不适用；无对应冲突检测或协商逻辑可述。

---

## 2) 依据

- **Memory Context 文档**：`docs/codex/07-codex-memory-context.md` 中仅描述 `drop_last_n_user_turns()` 为“回滚最近 N 个用户回合”，属于上下文/历史条目的裁剪，**未涉及对工作区文件的恢复或回滚**。
- 未发现“Session.revert”“Snapshot.revert”“文件回滚”“用户已编辑与源文件冲突”等与“revert 回滚 + 文件冲突”直接相关的设计或实现说明。

故基于现有文档，**视为无文件级 revert，进而无“用户已编辑与源文件冲突”的当前处理**。

---

## 3) 小结

| 项目                 | 当前情况                                         |
|----------------------|--------------------------------------------------|
| 文件级 revert 文档   | 未在现有文档中体现                               |
| 回滚能力             | 对话历史回退（如 drop_last_n_user_turns）        |
| 用户编辑 vs 源文件冲突 | 不适用（无文件回滚流程可依）                      |
| 冲突检测/提示        | 无相关描述                                       |

---

**一句话**：当前文档下 Codex 未体现文件级 revert，因此不存在“revert 时发现用户已编辑与源文件冲突”的现状处理说明。

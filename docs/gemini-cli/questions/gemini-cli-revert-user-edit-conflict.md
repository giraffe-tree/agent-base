# Gemini CLI：使用 revert 回滚时，用户已编辑与源文件冲突怎么处理？

本文基于现有文档说明 **Gemini CLI 当前**是否具备“revert 回滚时发现用户已编辑、与源文件冲突”的流程与处理。只讲现状，不讨论未来可如何实现。

---

## 1) 结论

**当前文档中未体现 Gemini CLI 具备用户可触发的、文件级的 revert/rollback 能力。**  
因此“revert 时发现用户已经编辑过、与源文件冲突怎么处理”的**文件冲突**场景，在现有文档范围内不适用；无对应冲突检测或协商逻辑可述。

---

## 2) 依据

- 在 `docs/gemini-cli` 下，与回滚/checkpoint 相关的描述仅出现在 `questions/gemini-cli-tool-call-concurrency.md` 中，且为并发/状态中的 `checkpoint?: string` 等字段，**未描述会话级或文件级的 revert/rollback 流程**。
- 未发现“Session.revert”“Snapshot.revert”“文件回滚”“用户编辑冲突”等与“revert 回滚 + 用户编辑冲突”直接相关的设计或实现说明。

故基于现有文档，**视为无文件级 revert，进而无“用户已编辑与源文件冲突”的当前处理**。

---

## 3) 小结

| 项目                 | 当前情况                                         |
|----------------------|--------------------------------------------------|
| 文件级 revert 文档   | 未在现有文档中体现                               |
| 用户编辑 vs 源文件冲突 | 不适用（无文件级 revert 流程可依）                |
| 冲突检测/提示        | 无相关描述                                       |

---

**一句话**：当前文档下 Gemini CLI 未体现文件级 revert，因此不存在“revert 时发现用户已编辑与源文件冲突”的现状处理说明。
